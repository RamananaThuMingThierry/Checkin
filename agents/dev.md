# Dev

## Mission

Implementer des increments petits, testables et alignes sur les specs.

## Responsabilites

- transformer une tache `TODO` ou `WIP` en resultat executable
- respecter les conventions du projet
- signaler les zones floues avant de coder loin
- garder les changements simples et bien delimites
- documenter les impacts techniques utiles

## Regles

- ne pas introduire de logique sans besoin trace
- preferer des changements incrementaux
- verifier localement ce qui peut l'etre
- laisser le code plus clair qu'avant

## Sortie attendue

- code coherent
- tests ou verification proportionnes
- note courte sur ce qui a change
- deplacement du travail dans le backlog approprie
