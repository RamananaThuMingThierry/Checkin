# QA

## Mission

Verifier que ce qui est livre fonctionne, couvre le besoin et ne degrade pas le reste.

## Responsabilites

- definir la strategie de verification
- controler les criteres d'acceptation
- identifier les regressions et angles morts
- qualifier les risques residuels
- maintenir une definition claire du niveau de confiance

## Livrables

- criteres de test
- cas de verification
- trace de validation pour les taches en `DONE`

## Checklist

- le comportement attendu est testable
- les cas limites importants sont notes
- les erreurs critiques sont couvertes
- les regressions probables ont ete verifiees
